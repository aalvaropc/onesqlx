# Authentication Flow — OneSQLx

## Overview

Authentication was generated by `phx.gen.auth` (Phoenix 1.8). It uses a
**scope-based** model where `@current_scope` (not `@current_user`) is the main assign.
The primary login method is **magic link** (passwordless), with password as a
secondary option.

## Key Files

### Plugs and Helpers

- `lib/onesqlx_web/user_auth.ex` — authentication plugs:
  - `fetch_current_scope_for_user/2` — loads user from session or remember-me cookie
  - `require_authenticated_user/2` — redirects to login if not authenticated
  - `on_mount` callbacks: `:mount_current_scope`, `:require_authenticated`, `:require_sudo_mode`

### LiveViews

| File | Route | Purpose |
|---|---|---|
| `live/user_live/registration.ex` | `/users/register` | Email registration |
| `live/user_live/login.ex` | `/users/log-in` | Login (magic link + password) |
| `live/user_live/confirmation.ex` | `/users/log-in/:token` | Magic link confirmation |
| `live/user_live/settings.ex` | `/users/settings` | Email and password changes |

### Controller

- `controllers/user_session_controller.ex` — handles POST login, POST update-password, DELETE logout

### Context

- `lib/onesqlx/accounts.ex` — business functions (registration, tokens, sessions)
- `lib/onesqlx/accounts/user.ex` — User schema with binary_id
- `lib/onesqlx/accounts/user_token.ex` — session and magic link tokens
- `lib/onesqlx/accounts/scope.ex` — Scope model that wraps the user
- `lib/onesqlx/accounts/user_notifier.ex` — email delivery

## Router — live_sessions

```
:current_user (public, optional auth)
├── /users/register
├── /users/log-in
└── /users/log-in/:token

:require_authenticated_user (protected)
├── /dashboard
├── /users/settings
└── /users/settings/confirm-email/:token
```

## Token Strategy

- **Session**: token in session cookie, reissued every 7 days
- **Remember-me**: cookie `_onesqlx_web_user_remember_me`, 14 days
- **Sudo mode**: requires recent authentication (10 minutes) for sensitive operations

## Golden Rule

Never use `@current_user`. Always use `@current_scope` and access the user with
`@current_scope.user`. Pass `current_scope` as the first argument to context functions.
